### 图片解析功能（新增）
- 使用 DeepSeek V4 视觉 API（2026年6月全量开放识图能力）
- API 端点：`https://api.deepseek.com/v1/chat/completions`，模型名 `deepseek-v4`
- OpenAI 兼容格式，图片用 base64 传入 content 数组
- 用户自行输入 API Key，存储在 localStorage，不经服务器
- 功能流程：上传图片 → AI 分析 → 返回12维度结构化关键词 → 匹配现有词库 → 一键应用到提示词选择器
- 图片自动压缩（长边限2000px，JPEG 0.85质量）避免超时
- 系统提示词要求 AI 返回 JSON 格式（dimensions/full_prompt/description）
- 客户端模糊匹配算法：精确匹配(100分) > 英文包含(80分) > 词汇匹配(60分)
- 入口：header 中的"图片解析"按钮，弹出全屏模态框

### 图片解析 API 格式修复 (2026-07-03 下午)
- **问题**：调用 DeepSeek API 报错 `unknown variant 'image_url', expected 'text'`
- **根因**：DeepSeek V4 的图片输入格式与 OpenAI 完全不同！
  - ❌ 错误格式（OpenAI风格）：content 数组内放 `{"type":"image_url","image_url":{...}}`
  - ✅ 正确格式（DeepSeek风格）：`image_data` 作为 message 的**兄弟字段**，content 为纯字符串
- **修复内容**：
  1. 模型名：`deepseek-v4` → `deepseek-v4-pro`（官方正确的模型ID）
  2. 请求体结构：content 从数组改为字符串，图片从 `content[].image_url` 移到顶层 `image_data` 字段
  3. Base64 编码：去掉 `data:image/jpeg;base64,` 前缀，只传纯 base64 字符串
  4. deep_thought：改为 `true`（深度思考模式对识图准确率提升显著）
  5. 提示词简化：去掉冗余的"you are an expert"角色设定，直接给分析指令

### 历史记录后重新上传的 Bug 修复 (2026-07-03 下午)
- **问题**：从历史记录加载解析结果后，点击"重新上传"选文件，预览不显示、开始解析按钮不启用
- **根因分析**（3个问题叠加）：
  1. `btnReupload` 点击时没有重置 `fileInput.value` — 浏览器缓存可能导致 `change` 事件不触发
  2. `handleImageFile` 没有 `img.onerror` / `reader.onerror` 处理 — Image 加载/解码失败则静默无反馈
  3. `viewHistoryItem` 将缩略图赋值给 `currentImageBase64` 并调用 `checkAnalyzeReady()` — 导致"开始解析"按钮错误地显示为可用（但缩略图太小无法用于分析），重新上传后状态混乱
- **修复**：
  1. 重新上传时清空 `fileInput.value = ''`
  2. `handleImageFile` 增加 try-catch 包裹 Canvas 操作，添加 `img.onerror` 和 `reader.onerror` 回调（显示 toast 提示 + console.error）
  3. 增加 DOM 元素空值检查（`previewImg`/`uploadZone`/`imagePreview`）
  4. `viewHistoryItem` 不再设置 `currentImageBase64`，显式禁用分析按钮，清除 `fileInput.value`
